# Model Evaluation & Validation Reference

Proper evaluation ensures that machine learning models generalize effectively to unseen data in production environments.

---

## 1. Classification Metrics Matrix

| Metric | Formula | Best Used For |
| :--- | :--- | :--- |
| **Accuracy** | $\frac{TP + TN}{TP + TN + FP + FN}$ | Balanced datasets ($50/50$ target split) |
| **Precision** | $\frac{TP}{TP + FP}$ | Minimizing False Positives (e.g., Spam Filter) |
| **Recall (Sensitivity)** | $\frac{TP}{TP + FN}$ | Minimizing False Negatives (e.g., Disease Detection) |
| **F1-Score** | $2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$ | Harmonic mean of Precision & Recall |
| **Balanced Accuracy** | $\frac{1}{2} \left( \frac{TP}{TP + FN} + \frac{TN}{TN + FP} \right)$ | Imbalanced datasets |
| **ROC-AUC** | Area under $TPR$ vs $FPR$ curve | Ranking ability across all classification thresholds |
| **PR-AUC** | Area under Precision vs Recall curve | Highly imbalanced datasets ($< 5\%$ positive class) |

---

## 2. Regression Metrics Matrix

| Metric | Formula | Properties |
| :--- | :--- | :--- |
| **Mean Absolute Error (MAE)** | $\frac{1}{n}\sum \|y_i - \hat{y}_i\|$ | Robust to outliers, intuitive units |
| **Mean Squared Error (MSE)** | $\frac{1}{n}\sum (y_i - \hat{y}_i)^2$ | Penalizes large errors heavily |
| **Root Mean Squared Error (RMSE)** | $\sqrt{\text{MSE}}$ | Same scale as target, sensitive to outliers |
| **R² Score (Coeff. of Determination)** | $1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$ | Percentage of target variance explained ($R^2 \le 1$) |

---

## 3. Threshold Optimization for Classification

By default, binary classification decision threshold is $T = 0.5$. In business applications, $T$ should be tuned to maximize business utility:

$$\text{Utility}(T) = TP(T) \cdot V_{TP} - FP(T) \cdot C_{FP} - FN(T) \cdot C_{FN}$$

Where:
- $V_{TP}$: Value generated by True Positive
- $C_{FP}$: Cost incurred by False Positive
- $C_{FN}$: Cost incurred by False Negative

---

## 4. Cross-Validation Schemes

### Stratified K-Fold
Preserves target class percentages across every fold. Crucial for classification problems.

### Group K-Fold
Ensures that all samples from a specific group (e.g., patient ID, store ID) stay entirely within a single fold to prevent data leakage across groups.

### Time-Based Holdout / TimeSeriesSplit
Prevents future information leakage into past predictions:

```
Fold 1: Train [t0..t1] ➔ Val [t1..t2]
Fold 2: Train [t0..t2] ➔ Val [t2..t3]
Fold 3: Train [t0..t3] ➔ Val [t3..t4]
```
